AOT risk areas (from research):
- `client/src/app/speaking/speaking.component.ts` — `tiles` and `comingSoon` arrays typed as implicit `any[]`
- Template accesses `tile.thumbnail`, `tile.svg`, `tile.url`, `tile.hasThumbnail`
- With implicit `any`, Angular 8 AOT compiler will NOT flag these (fullTemplateTypeCheck is not enabled)
- If NG error codes appear, they will be on the speaking component templates

Recovery pattern if ng build fails with NG errors on speaking component:
Add a `SpeakingTile` interface with all fields optional:
```typescript
interface SpeakingTile {
  title?: string;
  url?: string;
  thumbnail?: string;
  svg?: string;
  hasThumbnail?: boolean;
}
```
Type the arrays: `tiles: SpeakingTile[] = [...]` and `comingSoon: SpeakingTile[] = [...]`
