<resolution_scheme>
<!-- This is the resolution convention the executor MUST implement. -->

**Author convention (document in a code comment, not in a README):**

In a local post markdown file, reference images relative to the `assets/blog/` directory:

```markdown
![Diagram of the three layers](media/effecting-change-as-consultants/layers.png)
```

**Why this path shape:**
- Mirrors the physical storage (`client/src/assets/blog/media/{slug}/...`) minus the `assets/blog/` prefix, so authors never type boilerplate.
- Keeps the markdown portable: a different renderer or static-site generator that serves markdown from the same directory would resolve these correctly with no rewriting.
- Avoids SPA-routing breakage: a naive relative `src` would be resolved by the browser against `/posts/{slug}`, yielding `/posts/media/.../file.png` (404). The custom renderer rewrites to an anchored `assets/blog/...` path.

**Renderer behavior:**
- If `href` starts with `http://`, `https://`, `//`, `/`, or `data:` → leave untouched.
- Otherwise → emit `<img src="assets/blog/{href}" alt="{text}"{ title attr if present }/>`.
- Do not encode or re-escape — `marked` has already done its job on the `text` (alt).
</resolution_scheme>

The new renderer MUST:

1. Accept the three image arguments as `marked` provides them (in v9+: a single object `{ href, title, text }`; in older majors: positional `(href, title, text)`). Match the version installed.
2. Treat the href as absolute (pass through unchanged) when it matches any of:
   - starts with `http://` or `https://`
   - starts with `//` (protocol-relative)

3. Otherwise, prepend `assets/blog/` to the href. Do NOT add a leading slash — `assets/blog/...` is what `blog.service.ts` already uses for the markdown file itself, and is the form the site uses everywhere (`assets/blog/posts.json`, `assets/blog/{slug}.md`).
4. Emit a standard `<img>` tag: `<img src="{resolved}" alt="{text}" />`, and include `title="{title}"` when title is a non-empty string. Escape double-quote characters in `title` and `text` by replacing `"` with `&quot;` — do NOT pull in a new HTML-escape dependency; a one-line replace is sufficient for the alt/title surface here (content authors control the source).
5. Leave all other token rendering to `marked` defaults — do not override paragraph, heading, code, etc.

Place the renderer wiring at module load (top-level, alongside the existing `marked.setOptions` call) so it applies to every `renderMarkdown` invocation. Keep the exported signature `renderMarkdown(markdown: string): string` unchanged.
